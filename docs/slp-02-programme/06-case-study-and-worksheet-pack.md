# Case Study and Worksheet Pack

## Case Study 1: AI-Assisted Shortlisting

### Scenario

A public university department receives 800 applications for administrative posts. The HR team wants to use an AI-enabled shortlisting tool to rank candidates before human review.

### Learner Task

Prepare a governance review that addresses:

- Purpose of the tool.
- Candidate data used.
- Bias risks.
- Human review.
- POPIA considerations.
- Vendor questions.
- Recommendation to approve, approve with controls, defer or reject.

### Discussion Questions

- What should candidates be told?
- What data should not be used?
- How could historical hiring patterns affect the system?
- What evidence should be retained for audit purposes?

## Case Study 2: Employee Monitoring and Productivity Analytics

### Scenario

A manager wants to use AI-enabled productivity analytics to monitor remote workers. The tool analyses login times, application usage and message response patterns.

### Learner Task

Develop a risk map and control plan.

### Discussion Questions

- Is the monitoring proportionate?
- Could the tool misinterpret flexible work patterns?
- How should employees be consulted or informed?
- What human review should happen before action is taken?

## Case Study 3: Learning Recommendation System

### Scenario

An organisation wants an AI system to recommend training courses based on employee role, performance data and career aspirations.

### Learner Task

Design a governance approach that preserves fairness, privacy and employee agency.

### Discussion Questions

- Could the system reinforce existing inequality?
- What data is necessary?
- How can employees challenge or change recommendations?
- What success metrics should be monitored?

## Worksheet 2: POPIA and Responsible AI Checklist

| Area | Review Question | Evidence Needed |
|---|---|---|
| Purpose | Is the purpose specific and legitimate? | Use-case statement |
| Minimality | Is only necessary data used? | Data inventory |
| Information quality | Is the data accurate and current? | Data quality report |
| Openness | Are affected persons informed? | Notice or communication |
| Security | Are safeguards documented? | Security review |
| Automated decision-making | Could a person be significantly affected by an automated outcome? | Decision-flow map |
| Human oversight | Who can review or override the output? | Oversight procedure |
| Vendor control | What does the vendor process and store? | Vendor documentation |

## Worksheet 3: Bias and Fairness Review

| Bias Source | How It Could Appear | Possible Harm | Control |
|---|---|---|---|
| Historical data | Past decisions reflect inequality | Qualified candidates excluded | Review data and outcomes |
| Proxy variable | Neutral variable stands in for protected attribute | Indirect discrimination | Test and remove proxy |
| Poor labels | Performance labels are subjective | Unfair recommendations | Validate labels |
| Unequal access | Some groups have less digital access | Unequal measurement | Adjust interpretation |

## Worksheet 5: AI Incident Response Plan

| Incident | Immediate Action | Escalation | Evidence to Preserve | Follow-Up |
