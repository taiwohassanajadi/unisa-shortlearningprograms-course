# Curriculum and Weekly Learning Matrix

## Weekly Structure

The course is designed for 12 weeks. Each week includes guided study, applied workflow design, discussion and contribution to the final agentic workflow blueprint.

| Week | Study Unit | Topic | Learning Outcomes | Learning Activities | Assessment Link |
|---|---|---|---|---|---|
| 1 | AHRM04A | Orientation: agentic AI and HRM transformation | Explain the course purpose and identify HR workflows affected by agents | AI agent readiness self-audit, HR workflow discussion | Portfolio context note |
| 2 | AHRM04A | Chatbots, copilots, automation and AI agents | Distinguish AI tool types and levels of autonomy | Tool-type comparison, hype-risk discussion | Short reflection |
| 3 | AHRM04A | Automation versus augmentation | Analyse human agency and task suitability | Task triage, automation red-light/green-light exercise | Formative task 1 submission |
| 4 | AHRM04B | Workflow mapping and process discovery | Map current HR workflow steps, actors and handoffs | Process mapping workshop | Formative task 2 preparation |
| 5 | AHRM04B | Automation readiness and data access | Assess readiness, data quality, volume, risk and permissions | Readiness checklist, access-map activity | Readiness draft |
| 6 | AHRM04B | Failure points and exception handling | Identify breakdowns, exceptions and escalation needs | Failure-mode review, exception map | Formative task 2 submission |
| 7 | AHRM04C | Human-in-the-loop workflow design | Design approval gates and human review points | Oversight design clinic | Formative task 3 preparation |
| 8 | AHRM04C | Permissions, audit trails and agent boundaries | Map permissions, logs, controls and prohibited actions | Permission matrix, audit trail worksheet | Formative task 3 submission |
| 9 | AHRM04D | Pilot design for agentic HRM | Build a low-risk pilot proposal | Pilot canvas, risk register | Portfolio draft |
| 10 | AHRM04D | Vendor, security and governance review | Assess vendor claims, tool risk and governance requirements | Vendor questionnaire, governance review | Portfolio draft |
| 11 | AHRM04D | Evaluation, scaling and retirement | Define success metrics and criteria for scaling or stopping | Metrics workshop, scale/stop decision framework | Portfolio draft review |
| 12 | Capstone | Final workflow blueprint presentation | Present a defensible agentic HRM workflow blueprint | Peer presentation, final reflection | Final portfolio submission |

## Study Unit Outcomes

### AHRM04A - From Chatbots to AI Agents in HRM

Learners should be able to:

- Define chatbot, copilot, workflow automation, AI agent and multi-agent system.
- Explain differences between automation and augmentation.
- Identify HRM use cases where agentic AI may support work.
- Discuss maturity, hype risk and human agency concerns.

### AHRM04B - HR Workflow Mapping and Automation Readiness

Learners should be able to:

- Map current workflows, actors, inputs, outputs and handoffs.
- Identify task types suitable or unsuitable for automation.
- Assess readiness using volume, standardisation, data quality, risk and governance criteria.
- Identify exception paths and failure points.

### AHRM04C - Human-in-the-Loop Agent Workflow Design

Learners should be able to:

- Define agent roles, permissions and boundaries.
- Build approval gates and escalation points.
- Design audit trails and logs.
- Prevent over-permissioned or unsupervised agent actions.

### AHRM04D - Pilot, Evaluation, Governance and Scaling

Learners should be able to:

- Design a low-risk HR agent pilot.
- Define success, risk and employee-experience metrics.
- Review vendor and security considerations.
- Recommend scale, revise, pause or retire decisions.

## Workload Estimate

Recommended weekly workload:

- Guided reading: 2 hours.
- Learning material and recorded lecture: 1 hour.
- Practical workflow activity: 2 hours.
- Discussion or peer review: 1 hour.
- Portfolio development: 1 to 2 hours.

Estimated total workload: 84 to 96 hours.
