# Assessment Pack and Rubrics

## Assessment Overview

The course uses an applied workflow blueprint model. Each formative task builds toward the final Agentic HRM Workflow Blueprint.

| Assessment | Weight | Due | Output |
|---|---:|---|---|
| Formative Task 1 | 10% | Week 3 | HR workflow automation readiness assessment |
| Formative Task 2 | 15% | Week 6 | Current workflow and exception map |
| Formative Task 3 | 15% | Week 8 | Agent risk and permission map |
| Final Portfolio | 60% | Week 12 | Agentic HRM Workflow Blueprint |

## Formative Task 1: HR Workflow Automation Readiness Assessment

### Purpose

Learners assess whether an HRM or people-management workflow is suitable for agentic automation or AI-assisted orchestration.

### Instructions

Select one workflow. Examples include onboarding coordination, employee query triage, learning administration, policy retrieval, recruitment scheduling, case routing or routine HR reporting.

Submit a 1,000 to 1,200 word readiness assessment covering:

- Workflow description.
- Current pain point.
- Task type and autonomy level.
- Expected benefits.
- Data and system requirements.
- Risk level.
- Readiness rating.
- Recommendation: automate, augment, redesign first, or avoid.

### Marking Criteria

| Criterion | Weight |
|---|---:|
| Workflow and problem clarity | 25% |
| Automation readiness reasoning | 30% |
| Risk and data-awareness | 25% |
| Practical recommendation | 10% |
| Clarity and professionalism | 10% |

## Formative Task 2: Current Workflow and Exception Map

### Purpose

Learners map the current workflow, including handoffs, exceptions and failure points.

### Instructions

Submit a workflow map in table or diagram form. Include:

- Process steps.
- Actors.
- Systems or tools.
- Data used.
- Decisions.
- Handoffs.
- Exceptions.
- Failure points.
- Escalation needs.

Recommended length: 1,200 to 1,500 words plus table or diagram.

### Marking Criteria

| Criterion | Weight |
|---|---:|
| Current workflow completeness | 25% |
| Handoff and exception analysis | 25% |
| Data and system mapping | 20% |
| Failure point identification | 20% |
| Clarity and usability | 10% |

## Formative Task 3: Agent Risk and Permission Map

### Purpose

Learners define boundaries for a proposed HR AI agent.

### Instructions

Create a risk and permission map. Include:

- Agent role.
- Allowed actions.
- Prohibited actions.
- Data access.
- System access.
- Human approval gates.
- Escalation rules.
- Logging and audit trail needs.

Recommended length: 1,200 to 1,500 words.

### Marking Criteria

| Criterion | Weight |
|---|---:|
| Agent role and boundary clarity | 25% |
| Permission and data-access design | 25% |
| Human oversight and escalation | 25% |
| Audit trail and monitoring design | 15% |
| Professional structure | 10% |

## Final Portfolio: Agentic HRM Workflow Blueprint

### Purpose

Learners integrate the course learning into a practical blueprint for piloting an agent-assisted HRM workflow.

### Required Sections

1. Executive summary.
2. Workflow context and problem statement.
3. Current workflow map.
4. Automation readiness assessment.
5. Proposed agent-assisted workflow.
6. Data, system and permission map.
7. Human approval gates and escalation model.
8. Risk register.
9. Audit trail and monitoring plan.
10. Pilot implementation plan.
11. Success metrics.
12. Scale, revise, pause or retire decision criteria.

Recommended length: 4,000 to 5,500 words.

### Final Portfolio Rubric

| Criterion | Weight |
|---|---:|
| Workflow problem and readiness analysis | 15% |
| Current and future workflow design | 15% |
| Agent role, permission and boundary design | 15% |
| Human oversight and escalation model | 15% |
| Risk register and controls | 15% |
| Audit trail, monitoring and evidence | 10% |
| Pilot evaluation and scale/stop criteria | 10% |
| Professional structure and recommendation quality | 5% |

## AI Use in Assessment

Learners may use AI tools for brainstorming, workflow drafting or critique, but they must:

- Disclose how AI was used.
- Avoid uploading personal, employee, candidate or confidential organisational data.
- Verify all AI-assisted outputs.
- Take responsibility for final recommendations.
