# SLP 04: Agentic HRM and Workflow Automation for Managers

## Proposed Qualification Code

To be assigned by UNISA.

## Duration

12 weeks.

## Language Medium

English.

## Target Group

This programme is aimed at HR operations managers, service delivery managers, HR transformation teams, shared-services leaders, consultants, line managers and postgraduate learners interested in AI agents and workflow automation in HRM.

## Problem Being Solved

Organisations are moving from AI chatbots to AI agents that can plan steps, call tools, draft outputs and trigger actions across systems. In HRM this creates opportunities for faster service delivery, but also new risks: over-automation, unclear accountability, data exposure, bias, weak escalation paths and reduced human agency.

This SLP teaches learners how to identify HR workflows suitable for agentic AI, design human-in-the-loop controls, and build an automation readiness case before implementing agents.

## Purpose

The purpose of this programme is to teach practical skills for evaluating, designing and governing AI-agent and workflow-automation use cases in HRM.

## Learning Outcomes

On completion, learners should be able to:

- Explain the difference between chatbots, copilots, automation and AI agents.
- Identify HR workflows that may benefit from agentic automation.
- Map permissions, data access, handoffs and escalation points.
- Design human-in-the-loop controls for HR AI agents.
- Build a pilot proposal for an agentic HRM workflow.

## Study Units

### AHRM04A - From Chatbots to AI Agents in HRM

Content areas:

- Definitions: chatbot, copilot, workflow automation, agentic AI and multi-agent systems.
- HRM use cases: employee queries, onboarding, learning administration, recruitment coordination, policy retrieval and case triage.
- Automation versus augmentation.
- Human agency and worker preference.
- Current maturity and hype risks.

### AHRM04B - HR Workflow Mapping and Automation Readiness

Content areas:

- Process discovery and service blueprinting.
- Identifying repetitive, rules-based and judgment-heavy tasks.
- Data access, system integration and permission mapping.
- Failure points and exception handling.
- Readiness criteria: volume, risk, standardisation, data quality, governance and user trust.

### AHRM04C - Designing Human-in-the-Loop Agent Workflows

Content areas:

- Agent roles, permissions and boundaries.
- Human approval gates.
- Escalation design.
- Audit trails and logs.
- Prompt and tool governance.
- Preventing over-permissioned systems.

### AHRM04D - Pilot, Evaluation and Scaling

Content areas:

- Building a low-risk HR agent pilot.
- Success metrics: turnaround time, quality, employee satisfaction, risk incidents and cost.
- Change management and user training.
- Vendor evaluation.
- Governance for scaling.
- Continuous monitoring and retirement of unsafe workflows.

## Assessment

- **Formative task 1:** HR workflow automation readiness assessment.
- **Formative task 2:** Human-in-the-loop control design.
- **Formative task 3:** Agent risk and permission map.
- **Final portfolio:** Agentic HRM pilot proposal.

## Capstone Artefact

Learners produce an **Agentic HRM Workflow Blueprint** containing:

- Selected HR process.
- Current workflow map.
- Proposed agent-assisted workflow.
- Data and system access map.
- Human approval gates.
- Escalation rules.
- Risk register.
- Pilot evaluation plan.

## Evidence Base

Agentic AI research is still emerging, so this SLP should be positioned as a frontier course. The strongest current direction is not "replace HR teams with agents", but teach managers to evaluate where agents can safely augment service delivery while preserving human accountability and worker agency.
